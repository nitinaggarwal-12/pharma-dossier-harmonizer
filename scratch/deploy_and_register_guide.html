🚀 Pharma Dossier Harmonizer — Deployment & Registration Guide

This guide provides the complete step-by-step technical instructions to build and deploy the Pharma Dossier Harmonizer agent onto Vertex AI Agent Engine (Reasoning Engine), and register it inside the Gemini Enterprise Plus UI for corporate end-users.

📂 Section 1: Build & Deploy to Vertex AI Reasoning Engine

The ADK multi-agent orchestrator is packaged and deployed as a serverless Reasoning Engine on Google Cloud Platform (GCP).

1. GCP Infrastructure Prerequisites

Ensure your active gcloud identity has the Discovery Engine Admin, Vertex AI Administrator, and Storage Admin IAM roles.

Enable all mandatory GCP service APIs by executing:

gcloud services enable \
  aiplatform.googleapis.com \
  cloudresourcemanager.googleapis.com \
  discoveryengine.googleapis.com \
  documentai.googleapis.com \
  bigquery.googleapis.com

2. Staging ground GCS Bucket

The agent requires an active Cloud Storage bucket to store artifacts and support vector caches. Create your bucket if not already present:

gsutil mb -l us-central1 gs://pharma-dossiers-nitinagga-ge/

3. Deploying via the ADK CLI

The deployment packages your local pharma_agent/ source files (including agents tool-routing parameters, vector similarity schemas, and prompts) and registers them as a serverless Reasoning Engine:

# 1. Navigate to project root
cd /Users/nitinagga/Documents/pharma-dossier-ge

# 2. Activate virtual environment
source .venv/bin/activate

# 3. Deploy using the ADK CLI
adk deploy agent_engine \
  --project="nitinagga-ge" \
  --region="us-central1" \
  --display_name="Pharma Dossier Harmonizer" \
  ./pharma_agent

Expected Terminal Output:

Packaging pharma_agent directory...
Uploading codebase to GCS artifact registry...
Creating serverless Vertex AI Reasoning Engine instance...
✓ Reasoning Engine deployed successfully!
Resource ID: projects/528479452485/locations/us-central1/reasoningEngines/8971879092966129664

⚠️ IMPORTANT: Copy the Resource ID! You will need the exact projects/.../reasoningEngines/... string to register the agent inside Gemini Enterprise.

📂 Section 2: Register in Gemini Enterprise Plus UI

Once the agent is deployed as a Vertex Reasoning Engine, an administrator must register it so it appears natively as a secure chat option in Gemini Workspace.

1. Pre-requisites for Registration

  • Role Required: Discovery Engine Admin role on the GCP project.
  • Gemini Enterprise App: An active Gemini Enterprise app must be provisioned in your Google Workspace Admin console.

2. Step-by-Step Registration in Google Cloud Console

  1. Open the Google Cloud Console.
  2. Use the search bar to navigate to Gemini Enterprise / Agent management (or search Discovery Engine / Agent Builders).
  3. Click the Register Custom Agent button on the top menu.
  4. Select Vertex AI Reasoning Engine (ADK) as the agent type.
  5. Fill out the Agent Details:
    • Project ID: nitinagga-ge
    • Location: us-central1
    • Agent Engine Resource ID: Paste the exact Resource ID copied from your deployment terminal (e.g., projects/528479452485/locations/us-central1/reasoningEngines/8971879092966129664).
    • Display Name: Pharma Dossier Harmonizer
    • Description: Expert pharmaceutical assistant for regulatory affairs, compliance, delta analysis, and eCTD Module 3 shell drafting.
  6. Click Next.

3. Configure Role-Based Visibility

To prevent unauthorized access to highly sensitive compound dossiers, assign visibility to specific Active Directory / Workspace groups:

  • Regulatory Affairs Group ➡️ Viewer / User Access.
  • Quality Assurance Group ➡️ Viewer / User Access.
  • Labeling & Prescribing Group ➡️ Viewer / User Access.
  • IT & System Admins ➡️ Full Admin / Management Access.

Click Save and Register.

The Pharma Dossier Harmonizer will now instantly populate as an active agent in the side panel for all authorized users inside their Gemini Workspace Chat!

📂 Section 3: Post-Deployment Conversational Smoke Tests

To confirm that the deployed reasoning engine is fully operational and actively calling its GCS, Document AI, and Vector RAG backend tools, log into Gemini Enterprise as an end-user and run these 3 smoke tests:

🧪 Smoke Test 1: GCS Folder & Naming Casing Scan

User Prompt:

Use your verify_ectd_packages tool to scan GCS folders and audit our files for casing gaps.

Expected Agent Response:

  • Compliance Flag: Displays 🔴 Non-Compliant due to spaces and uppercase letters.
  • File List: Lists scanned GCS files, including COCT_MT150007UV.xsd and eCTD 3.2.2 - EU M1 Validation Criteria v8.2.xlsx.

🧪 Smoke Test 2: Scientific Quality RAG Search

User Prompt:

Compare the draft dossier m3-dp-pharma-dev - working.md against M4Q_R1_Guideline.pdf quality standards and identify stability gaps. Cite exact page numbers.

Expected Agent Response:

  • ICH Citation: Verbatim citation of M4Q_R1_Guideline.pdf Page 17 (Section 3.2.S.7.1).
  • Gaps Flagged: Reports that the draft lacks mandatory tabulated 6-month accelerated stability potency figures under stress conditions.

🧪 Smoke Test 3: Safety Warning Boxed Labels Harmonization

User Prompt:

Harmonize the safety warnings in my staged dossier against DailyMed boxed warnings for Adalimumab.

Expected Agent Response:

  • Warnings Found: Identifies mandatory boxed warnings for Serious Infections and Malignancies.
  • Gap Finding: Flags that the staged dossier lacks these pediatric boxed warnings.
